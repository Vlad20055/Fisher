Роли пользователей системы:
1 Администратор компании (Admin)
    1.1 Сотрудник компании с максимальными правами.
    1.2 Управляет учётными записями всех менеджеров компании и сотрудников магазинов.
    1.3 Имеет доступ ко всем функциям менеджера компании.

2 Менеджер компании (Manager)
    2.1 Работник компании-производителя рыболовных товаров.
    2.2 Имеет полный доступ к управлению товарами и просмотру финансовых поступлений.
    2.3 Не может управлять учётными записями других пользователей.

3 Менеджер магазина (Client)
    3.1 Сотрудник магазина-покупателя.
    3.2 Может просматривать каталог товаров и формировать заказы.
    3.3 Видит только историю заказов своего магазина.


Требования к функциональности:
1 Управление пользователями и регистрация (Для Администратора компании)
    1.1 Регистрация менеджера компании:
        Менеджер компании должен иметь возможность самостоятельно зарегистрироваться, указав (логин, пароль, ФИО). Учётная записьт создаётся в статусе "неактивна".

    1.2 Подтверждение регистрации менеджера компании:
        Администратор должен:
        1.2.1 Видеть список всех зарегистрированных менеджеров компании.
        1.2.2 Иметь кнопку "Подтвердить" для активации учётной записи менеджера компании.
        1.2.3 Иметь кнопку "Удалить" для удаления учётной записи менеджера компании.

    1.3 Регистрация менеджера магазина:
        Менеджер магазина должен иметь возможность самостоятельно зарегистрироваться, указав данные магазина (Название, Адрес, ИНН) и свои данные (ФИО, логин, пароль). Учётная запись создаётся в статусе "неактивна".

    1.4 Подтверждение регистрации менеджера магазина:
        Администратор должен:
        1.4.1 Видеть список всех зарегистрированных менеджеров магазинов.
        1.4.2 Иметь кнопку "Подтвердить" для активации учётной записи менеджера магазинов.
        1.4.3 Иметь кнопку "Удалить" для удаления учётной записи менеджера магазинов.

2 Управление товарами (Для Менеджера компании и Администратора)
    2.1 Просмотр списка товаров:
        Система должна предоставлять возможность просмотра списка всех товаров с их атрибутами (ID, Название, Тип, Количество на складе, Цена за единицу).

    2.2 Добавление нового товара:
        Система должна предоставлять форму для добавления нового товара с указанием его названия, типа, количества на складе и цены.

    2.3 Редактирование существующего товара:
        Система должна позволять изменять атрибуты существующего товара (название, тип, количество на складе, цену).

    2.4 Удаление товара:
        Система должна предоставлять функцию удаления товара из системы.

3 Управление заказами (Для Менеджера магазина)
    3.1 Просмотр каталога товаров:
        Система должна предоставлять Магазину возможность просматривать каталог всех доступных товаров Поставщика (Название, Тип, Цена). Количество на складе может быть скрыто.

    3.2 Создание заказа:
        Система должна позволять Менеджеру магазина выбирать товары из каталога, указывать количество для каждого и формировать заказ.
        Система должна автоматически рассчитывать общую сумму заказа.
        Примечание: Формирование заказа должно уменьшать количество доступного товара на складе компании.

    3.3 Просмотр истории заказов:
        Система должна предоставлять Магазину возможность просматривать историю своих собственных заказов с деталями (ID заказа, Дата, Состав заказа, Общая сумма, Статус).

4 Финансы (Для Менеджера компании и Администратора)
    4.1 Просмотр счёта компании:
        Система должна отображать общую сумму всех полученных от магазинов платежей.
        Примечание: На MVP интеграция с реальными платёжными системами не требуется. Достаточно виртуального счёта, который увеличивается на сумму подтверждённого заказа.

5 Журналирование действий (Для Администратора компании)
    5.1 Ведение журнала действий:
        Система должна автоматически записывать в журнал все критически важные действия пользователей.
        Записываемые действия должны включать:
            5.1.1 Добавление, редактирование, удаление товара.
            5.1.2 Создание нового заказа.
            5.1.3 Действия администратора по управлению пользователями (подтверждение, удаление).
        Каждая запись в журнале должна содержать:
            5.1.4 Дата и время события.
            5.1.5 Роль и логин пользователя, выполнившего действие.
            5.1.6 Тип выполненного действия.
    
    5.2 Просмотр журнала действий:
        Администратор должен иметь возможность просматривать весь журнал действий.
